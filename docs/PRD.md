# 產品需求文件 (PRD) - 個人記帳簿

## 1. 專案概述
- **背景與動機**：現代人生活忙碌，常常不清楚資金的去向，導致過度消費或是難以掌握積蓄狀況。本系統旨在提供一個簡單易用的介面，幫助使用者輕鬆記錄每一筆收入與支出，清楚掌握自身財務狀況。
- **目標用戶**：所有人，尤其是希望養成記帳習慣、有初步理財與花費追蹤需求的大眾。
- **核心價值主張**：以最直覺的方式記錄日常收支，並透過分類統計與跨月對比，協助使用者輕鬆且有效地管理個人財富。

## 2. 功能需求
以下為系統主要功能及其對應的使用者故事：

1. **記錄收入與支出**
   - 作為 **使用者**，我希望 **能夠明確新增一筆收入或支出的紀錄（包含金額與備註說明）**，以便 **詳實追蹤我每一筆資金的進出狀況**。
   
2. **顯示餘額統計**
   - 作為 **使用者**，我希望 **能在進入系統時直觀地看到當前的總餘額與本月收支加總**，以便 **快速了解我目前可用的資金概況**。
   
3. **記錄每筆明細資料日期**
   - 作為 **使用者**，我希望 **在記帳時可以指定或自動帶入交易當天的日期**，以便 **未來回溯或依時間軸查詢整理時具有確切的時間依據**。
   
4. **將一個月內所有資料進行類別區分**
   - 作為 **使用者**，我希望 **能為收支紀錄設定不同的類別（如：餐飲、交通、薪水等），並觀看單一月份內各類別的花費總和**，以便 **精準分析我的主要花費落在哪個生活面向**。
   
5. **可以比較這個月和上個月消費差異**
   - 作為 **使用者**，我希望 **能看到本月與上個月的總支出差異與趨勢比對**，以便 **警惕自己是否有花費超標的情況，或檢視節流的成效**。

## 3. 非功能需求
- **技術限制**：
  - 後端：Python Web 框架 (Flask)
  - 前端：HTML + Jinja2 模板引擎與原生 CSS / JS
  - 資料庫：SQLite (確保開發與部署輕量化)
- **效能與安全考量**：
  - 基礎的防呆機制：例如金額不可為負數（或由前端擋下）、必填攔位檢查。
  - 確保月度或跨月統計計算的查詢效能，在多筆資料下仍可於 1 秒內回傳畫面。
  - 具備適配多裝置的響應式網頁設計（RWD），讓使用者能透過手機順利記帳。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 基本的收支明細 CRUD（新增、讀取/列表、修改、刪除）。
  - 對每筆紀錄可指派日期與預設的收支分類。
  - 計算並顯示總結餘、當月總收入與當月總支出。
  - 照類別區分的單月費用加總清單。
  - 本月與上月支出的簡單數字比較。
- **Should Have (應該有)**：
  - 使用者能自訂收支類別。
  - 清晰的版面互動微動畫與現代化的 UI 外觀。
- **Nice to Have (加分項)**：
  - 圖表化呈現（例如使用 Chart.js 呈現花費佔比圓餅圖或趨勢長條圖）。
  - 多帳號登入與個人化資料隔離機制。
  - CSV 匯出功能。
